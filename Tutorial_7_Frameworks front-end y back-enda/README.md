## Qué es front-end y back-end?

## 1. Front-end

Frontend es la parte de un programa o dispositivo a la que un usuario puede acceder directamente. Son todas las tecnologías de diseño y desarrollo web que corren en el navegador y que se encargan de la interactividad con los usuarios.

Conocimientos Necesarios:
1. HTML 
1. CSS
11. Javascript (*)
>(*) Javascript puede apoyarse de algunos framework, librrias o preprocesadores. EJ: 
>* React
>* Vue
>* Angular
>* Svelte
>* Bootstrap
>* Foundation
>* Sass
>* Less
>* Stylus
>* PostCSS

## 2. Back-End

Backend es la capa de acceso a datos de un software o cualquier dispositivo, que no es directamente accesible por los usuarios, además contiene la lógica de la aplicación que maneja dichos datos. El Backend también accede al servidor, que es una aplicación especializada que entiende la forma como el navegador solicita cosas.

Conocimientos Necesarios
1. Lenguajes de Programacion para Backend (*)

>(*) Estos tambien pueden apoyarse en frameworks, tales como:
>* Django
>* Flask
>* Express.js
>* Laravel
>* Symphony Framework
>* Ruby on Rails
>* `APS.Net`

## Principales diferencias entre front/back-end

| Diferencias          | Front-end                                                        | Back-End                                                                                              |
|----------------------|------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------|
| Enfoque              | Casi exclusivamente al usuario final                             | Principalmente para los desarrolladores. Rara vez el usuario final tiene un acceso muy limitado.      |
| Rol                  | Aspectos Visuales                                                | Todo lo que ocurre en Segundo Plano                                                                   |
| Desarrolladores      | Que la interfaz sea lo más legible posible para el Usuario Final | Asegurarse que los datos y sistemas solicitados por el software funcione de manera eficiente y fiable |
| Elementos esenciales | HTML, CSS, Javascript                                            | Java, Ruby, Python, PHP, .Net, entre otros                                                            |

## Lenguajes, Frameworks y Librerias

### Definiciones Importantes

1. FRAMEWORK

    >Es un conjunto de herramientas, librerías y buenas prácticas para resolver una *serie de problemas*.
    >> **Obs**: En Desarrollo Backend, *todos* los frameworks tienen la principal tarea de crear un servidor **web**.

1. LIBRERÍA

    >Son *fragmentos de código* creados por terceros que **solucionan un problema en específico** con el fin de *extender* las funcionalidades del código.

### ¿Cómo escoger el framework o libería correcta para mi proyecto?

Para escoger el framwork(librería) correcto(a), se ofrecen **7 puntos clave**

* Disponibilidad y calidad de la documentación
* Tamaño de la comunidad
* Issues abiertos en GitHub

    Que tan **activa** es la comunidad.

* Problemas que puede solucionar
* Flexibilidad
* Complejidad
* Compatibilidad con las otras herramientas que usarás
